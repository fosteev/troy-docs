# Game Overview

## Концепция

Troy — мобильная геолокационная RPG. Игрок перемещается в реальном мире, на карте вокруг него появляются монстры. Чтобы вступить в бой, нужно физически подойти к точке на карте. Бой проходит в реальном времени с системой скиллов и кулдаунов. За победу — опыт и лут.

## Core Loop

```
1. Открыть приложение → карта с позицией игрока
2. Увидеть монстров в радиусе видимости
3. Физически подойти к монстру (геолокация)
4. Начать бой (реалтайм, скиллы + автоатака)
5. Победить → получить XP + лут
6. Уровень вырос → новые скиллы, очки атрибутов
7. Экипировать лут → стать сильнее
8. Повторить с более сильными монстрами
```

## Основные сущности

### User (аккаунт)

Учётная запись игрока. Содержит:
- Email, пароль (хеш)
- JWT-авторизация
- Верификация email
- Список персонажей

### Character (персонаж)

Игровая сущность, привязанная к User. Содержит:
- Класс (Warrior / Mage)
- Уровень (1–30)
- Базовые атрибуты (Strength, Intelligence, Stamina, Agility, Spirit)
- Распределённые свободные очки
- Инвентарь и экипировка
- Текущие HP / классовый ресурс (Rage для Warrior, Mana для Mage)
- Позиция на карте (геолокация)

Один User может иметь несколько Character. На карте активен только один персонаж в каждый момент.

## Основные принципы дизайна

1. **Простота входа** — 2 класса, понятные механики, не перегружать новичка
2. **Глубина прокачки** — свободные очки атрибутов дают разные билды одного класса
3. **Физическое перемещение** — ходьба как основная механика, anti-cheat по скорости
4. **Честная боёвка** — результат боя зависит от уровня, шмота и умения использовать скиллы, а не от доната
5. **Расширяемость** — система атрибутов и скиллов позволяет добавлять новые классы без переделки ядра
